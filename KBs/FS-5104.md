# Knowledge Base Entry: Wrong Wholesale Role Prices Shown When Price Caching Enabled With Flatsome Theme

## 🎯 Executive Summary
**The Issue:** When the Wholesale Order Form's "Show Variations Individually" option is enabled alongside Wholesale price caching, the Flatsome theme causes the wrong wholesale role's prices to be displayed on the frontend (shop pages and order form) after a new user is added to any wholesale role — clearing the cache temporarily fixes it but the issue recurs.
**The Fix:** Update Wholesale Prices Premium to v2.0.6, which contains the permanent fix; as a temporary workaround, disable either "Show Variations Individually" in the order form or disable Wholesale price caching (Wholesale > Settings > Cache).

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** wrong wholesale prices showing, incorrect wholesale role pricing, price caching bug, prices showing for wrong role, cache clearing fixes prices temporarily, Flatsome theme wholesale pricing, Show Variations Individually wrong prices, price flips after adding user
>
> **Steps to solve:**
> 1. Confirm the customer is using the Flatsome theme and has "Show Variations Individually" enabled in the Wholesale Order Form settings AND has Wholesale price caching enabled (Wholesale > Settings > Cache).
> 2. Ask the customer to update Wholesale Prices Premium to v2.0.6 or later — this version contains the permanent fix.
> 3. After updating, go to Wholesale > Settings > Cache and click "Clear Cache" to ensure stale cached prices are removed.
> 4. If the customer cannot update immediately, offer these two temporary workarounds (either one stops the issue): (a) Disable "Show Variations Individually" in the Order Form editor, OR (b) Disable Wholesale price caching at Wholesale > Settings > Cache.
> 5. Confirm the fix by logging in as a wholesale user, checking prices, then creating a new backend user in any wholesale role and refreshing the frontend — prices should remain correct.

## 💡 Internal Insights (If applicable)
- The bug is specific to the combination of: Flatsome theme + Wholesale price caching enabled + "Show Variations Individually" active in the Order Form. Removing any one of those three factors prevents the issue.
- The checkout/cart always showed correct prices; only the frontend product/shop pages and order form were affected.
- Kinsta's nginx proxy cache is a separate layer and was definitively ruled out as a contributing factor after the InstaWP copy test (Kinsta mu-plugin removed, issue persisted only with Flatsome).
- Fixed in Wholesale Prices Premium v2.0.6.
