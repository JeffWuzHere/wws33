# Knowledge Base Entry: Fetching Wholesale Price Programmatically for Third-Party Plugin Integration

## 🎯 Executive Summary
**The Issue:** A third-party currency display plugin was using the regular (retail) price as the base for currency conversion instead of the wholesale price when a wholesale customer was logged in, resulting in incorrect converted prices shown on the product page.
**The Fix:** Wholesale Suite provides a PHP API function (`WWP_Wholesale_Prices::get_product_wholesale_price_on_shop_v3`) that third-party plugins or custom code can call to retrieve the correct wholesale price for the current user's role, which can then be passed to any currency conversion or display logic.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** fetch wholesale price in code, get wholesale price programmatically, third-party plugin wholesale price integration, currency conversion using wholesale price, wholesale price PHP function, WWP API, wholesale price for logged-in user
>
> **Steps to solve:**
> 1. Explain that Wholesale Suite does not have a direct official integration with the third-party plugin, but provides a PHP API that the third-party plugin's developer (or a developer working on custom code) can use.
> 2. To get the current logged-in user's wholesale role:
>    ```php
>    global $wc_wholesale_prices;
>    $currentUserRole = $wc_wholesale_prices->wwp_wholesale_roles->getUserWholesaleRole();
>    ```
> 3. To retrieve the wholesale price of a product for that role:
>    ```php
>    global $product;
>    $productID = $product->get_id();
>    $wholesaleProduct = WWP_Wholesale_Prices::get_product_wholesale_price_on_shop_v3( $productID, $currentUserRole );
>    $wholesalePrice = $wholesaleProduct['wholesale_price'];
>    ```
> 4. Pass `$wholesalePrice` to the third-party plugin's currency conversion function instead of the regular WooCommerce price.
> 5. If the customer needs to loop through all wholesale roles (e.g., for a display that shows prices per role), use `getAllRegisteredWholesaleRoles()` and iterate:
>    ```php
>    $wholesale_roles = $wc_wholesale_prices->wwp_wholesale_roles->getAllRegisteredWholesaleRoles();
>    foreach ( $wholesale_roles as $wholesale_role => $data ) {
>        $wholesale_price_data = WWP_Wholesale_Prices::get_product_wholesale_price_on_shop_v3( $productID, array( $wholesale_role ) );
>        $wholesale_sale_price_data = WWPP_Wholesale_Prices::get_product_wholesale_sale_price( $productID, array( $wholesale_role ) );
>    }
>    ```
> 6. Share the code with the third-party plugin's support team or a developer to implement the integration on their end.

global $wc_wholesale_prices;
$currentUserRole = $wc_wholesale_prices->wwp_wholesale_roles->getUserWholesaleRole();

$productID = $product->get_id();

## 💡 Internal Insights (If applicable)
- `WWP_Wholesale_Prices::get_product_wholesale_price_on_shop_v3()` is the primary API function for fetching wholesale prices in Wholesale Suite. It accepts a product ID and an array of wholesale role slugs.
- `WWPP_Wholesale_Prices::get_product_wholesale_sale_price()` (note the WWPP prefix — requires Wholesale Prices Premium) retrieves the wholesale sale price if one is set.
- `getUserWholesaleRole()` returns an array of the current user's wholesale roles; pass this directly as the second argument to `get_product_wholesale_price_on_shop_v3`.
- A feature request has been raised internally to improve compatibility/integration with third-party display plugins that modify price output.
- This pattern applies to any third-party plugin that reads WooCommerce product prices and needs to be wholesale-aware.
