# Knowledge Base Entry: Quantity Box Not Working for Wholesale Customers on Variable Products

## 🎯 Executive Summary
**The Issue:** Wholesale customers cannot increase the quantity of a variable product variant using the +/- quantity box — the number does not change when clicked — while non-wholesale customers are unaffected.
**The Fix:** The conflict is caused by the **Blocksy Companion plugin's Floating Cart** feature; disabling "Floating Cart" under Blocksy > Companion > Extension > Shop Extra resolves the issue immediately.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** quantity box not working, quantity not changing for wholesale customers, can't increase quantity on variable product, wholesale role quantity selector broken, Blocksy theme quantity issue
>
> **Steps to solve:**
> 1. Confirm the symptom is role-specific: the quantity box works for regular customers but fails for wholesale-role customers on variable products.
> 2. Ask if the customer is using the **Blocksy theme** with the **Blocksy Companion plugin** installed.
> 3. If yes: navigate to **Blocksy > Companion > Extensions > Shop Extra** and locate the **Floating Cart** option.
> 4. Disable the **Floating Cart** feature and save.
> 5. Test again as a wholesale customer — the quantity box should now work correctly.
> 6. Advise the customer to contact **Blocksy support** with the following detail: "The `floating-cart.js` file is being called multiple times in the browser console's network tab for wholesale role customers, but only once for non-wholesale customers — this is causing the quantity box to fail."
