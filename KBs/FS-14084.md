# Knowledge Base Entry: Wholesale Price Not Displaying on Bundle Product Page (Shows Retail Instead)

## 🎯 Executive Summary
**The Issue:** On WooCommerce bundle product single pages, the "Your Price" label shows the retail price instead of the wholesale price for logged-in wholesale customers — even though the wholesale price applies correctly in the cart.
**The Fix:** Add a custom PHP snippet to the theme's `functions.php` (or via WP Code plugin) that forces the wholesale price data to render on the single bundle product page; a confirmed bug in Wholesale Prices Premium where wholesale price data is missing from the bundle single product page output.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** bundle product showing retail price, Your Price showing wrong price on bundle page, wholesale price correct in cart but wrong on product page, bundle single product page price incorrect, WooCommerce Bundles wholesale price display
>
> **Steps to solve:**
> 1. Confirm the symptom: wholesale customer logs in, navigates to a bundle product single page — "Your Price" shows retail price. Cart/checkout shows the correct wholesale price. This is a confirmed bug in WWPP.
> 2. As a workaround until an official fix is released, add the following snippet to the **active theme's `functions.php`** or via the **WP Code plugin**: https://pastebin.com/HZnBRXPp
> 3. If the site uses a **child theme**, add the snippet to the child theme's `functions.php` — not the parent. If the snippet works on the parent theme but not the child theme, the child theme likely has conflicting code. In that case, advise the customer to check their child theme's `functions.php` for overrides affecting WooCommerce price hooks and to contact their theme developer.
> 4. Take a full site backup before adding any snippet.
> 5. After adding the snippet, verify: log in as a wholesale customer, visit a bundle product page, confirm "Your Price" now displays the correct wholesale price.
> 6. Note: if the bundle product has no **regular/retail price** set in the General tab, the wholesale price label may not display at all. Set at least a regular price on the bundle so the wholesale price context appears on the frontend.

## 💡 Internal Insights (If applicable)
- This is a confirmed bug (GitHub issue logged) — bundle single product page does not render wholesale price data in some configurations. The snippet at https://pastebin.com/HZnBRXPp is the approved workaround until a core fix is released.
- Child themes with custom WooCommerce template overrides or price-filter hooks can block the snippet from taking effect. Always test snippet on the parent theme first to isolate whether the issue is theme-side or plugin-side.
- If the bundle product has no regular price set in WooCommerce, the wholesale price label will not appear on the frontend regardless of the snippet — a regular price must be set on the bundle for the wholesale price display to render.
