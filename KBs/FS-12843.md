# Knowledge Base Entry: WWLC v2.0.4 — Approval Email Not Sending and User Email Wiped When Approving via Edit Panel

## 🎯 Executive Summary
**The Issue:** In WooCommerce Wholesale Lead Capture v2.0.4, approving a lead via the React UI's slide-out edit panel (not the quick-approve button) causes the user's email address to be wiped from the database. This prevents the approval email from sending and instead fires a "Email Changed to [blank]" notification to the customer.
**The Fix:** The agent confirmed the bug is replicable. Bugs were raised with developers. Until an official fix is released, a `wp_pre_insert_user_data` filter in `functions.php` that preserves the existing email when an empty one is submitted serves as a temporary workaround.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** wwlc approval email not sending, lead approval email missing, email changed notification instead of approval, user email wiped on approval, wholesale lead capture react UI approval bug, approve lead edit panel email blank, wwlc v2.0.4 bug
>
> **Steps to solve:**
> 1. Confirm the customer is on WWLC v2.0.4 and is approving leads via the **edit panel** (slide-out modal), not the quick-approve button. The quick-approve path does not trigger this bug.
> 2. **Root cause:** The React edit panel's REST API call to `PATCH/PUT wwlc/v1/leads/{user_id}` omits `wwlc_email` from the payload. This causes `wp_update_user()` to be called with `user_email => ''`, which wipes the email. The WWLC approval email trigger then receives a `WP_User` with no email and silently fails. WordPress fires an "Email Changed" notice to the (now-blank) address.
> 3. **Temporary workaround** (apply until an official fix is released): Add a `wp_pre_insert_user_data` filter to `functions.php` or WPCode that detects when an empty email is being submitted for an existing user and preserves the existing email:
>    ```php
>    add_filter( 'wp_pre_insert_user_data', function( $data, $update, $user_id ) {
>        if ( $update && empty( $data['user_email'] ) && $user_id ) {
>            $existing = get_userdata( $user_id );
>            if ( $existing ) {
>                $data['user_email'] = $existing->user_email;
>            }
>        }
>        return $data;
>    }, 10, 3 );
>    ```
> 4. Check whether an updated WWLC version has been released that includes the official fix before applying the workaround to a production site.
