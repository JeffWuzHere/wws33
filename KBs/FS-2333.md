# Knowledge Base Entry: Wholesale Order Form: $0 Price Products Not Showing and Custom Cart Error Messages

## 🎯 Executive Summary
**The Issue:** Products with a wholesale price of $0.00 did not appear in the Wholesale Order Form when "Only Show Wholesale Products to Wholesale Customers" was enabled; a separate issue was that WooCommerce cart error messages had messy formatting with unwanted quotation marks around product names.
**The Fix:** (1) Update Wholesale Prices to v2.2.4+ and Wholesale Prices Premium to v2.0.6+ which include a bug fix for $0 wholesale price visibility. As a temporary workaround, disable "Only Show Wholesale Products to Wholesale Customers" in Wholesale > Settings > General. (2) Use a `gettext` filter snippet to customize the WooCommerce stock error message text.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** zero price wholesale product not showing, free product wholesale, $0 not visible on order form, order form missing products, wholesale price zero hidden, cart error message customize, stock error message format, not enough stock message
>
> **Steps to solve (Zero price products not showing in Order Form):**
> 1. Confirm the customer has "Only Show Wholesale Products to Wholesale Customers" enabled under Wholesale > Settings > General.
> 2. As an **immediate workaround**: disable that setting temporarily.
> 3. For a permanent fix: update **Wholesale Prices Base** to v2.2.4+ and **Wholesale Prices Premium** to v2.0.6+. These releases include a bug fix specifically for missing products when wholesale price is set to 0. Note: the fix in v2.2.4 addresses the shop page visibility; the Order Form fix was included in WWPP v2.0.6.
> 4. After updating, re-enable "Only Show Wholesale Products to Wholesale Customers" and verify the $0-priced products now appear in the Order Form.
> 5. Refer to: https://wholesalesuiteplugin.com/kb/how-to-give-zero-wholesale-price-or-free-product-wholesale-customers-only/ for the recommended approach to free wholesale products.
>
> **Steps to solve (Customize WooCommerce stock error messages):**
> 1. Add the following snippet to the theme's `functions.php` or via the WP Code plugin, updating the message text as needed:
> ```php
> function wws_modify_cart_error_messages($translated_text, $text, $domain) {
>     switch ($text) {
>         case 'You cannot add that amount of "%1$s" to the cart because there is not enough stock (%2$s remaining).':
>             return 'Sorry, we only have %2$s units of "%1$s" available.';
>         case 'You cannot add that amount to the cart &mdash; we have %1$s in stock and you already have %2$s in your cart.':
>             return 'Sorry, you already have %2$s in your cart and we only have %1$s in stock.';
>         default:
>             return $translated_text;
>     }
> }
> add_filter('gettext', 'wws_modify_cart_error_messages', 20, 3);
> ```
> 2. The `%1$s` placeholder is the product name; `%2$s` is the stock quantity. Adjust the wording as preferred.

## 💡 Internal Insights (If applicable)
- The $0 wholesale price + "Show Wholesale Products Only" bug affected both the shop page and the Order Form, but the fixes shipped in separate releases: shop page fixed in Wholesale Prices Base v2.2.4, Order Form fixed in Wholesale Prices Premium v2.0.6.
- When a customer uses $0 wholesale pricing for free B2B promotional items, the recommended approach is the KB workaround, not a literal $0 price in the wholesale price field, as the plugin's visibility logic historically treated $0 as "no price set."
