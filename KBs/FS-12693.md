# Knowledge Base Entry: Wholesale Payments Invoice List Empty in WP Admin Despite Successful Stripe Transactions

## 🎯 Executive Summary
**The Issue:** With a NET30 Payment Terms rule configured in Wholesale Payments, Stripe receives the transaction data and generates invoices correctly on the Stripe side, but the invoice list inside the WP Admin Wholesale Payments dashboard remains empty — no invoices are visible.
**The Fix:** This was a confirmed bug fixed in Wholesale Payments v1.0.6. Updating to v1.0.6 or later resolves the empty invoice list and invoices populate correctly with payment links.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** wholesale payments invoices empty, WPAY invoice list blank, no invoices showing wholesale payments, net30 invoice not generating, invoice section empty, stripe invoice generated but not in wp-admin, wholesale payments v1.0.6
>
> **Steps to solve:**
> 1. Confirm the customer is using Wholesale Payments with NET30 (or similar payment terms) and that Stripe is receiving the data correctly but invoices are not appearing in WP Admin.
> 2. This is a known bug fixed in **Wholesale Payments v1.0.6**. Update the plugin via WP Admin > Plugins > Updates, or download manually from https://wholesalesuiteplugin.com/my-account.
> 3. After updating, the invoice list in the Wholesale Payments admin section should populate correctly with payment links.
> 4. Take a backup before updating, as a general precaution.

## 💡 Internal Insights (If applicable)
- The Stripe-side data was always being transmitted correctly — the bug was purely in the WP Admin invoice list not reading/displaying the data from Stripe.
- Wholesale Payments does not currently support manual order creation with wholesale pricing applied natively. Workaround: use the "User Switching" plugin (https://wordpress.org/plugins/user-switching/) to temporarily switch to the wholesale customer account and place the order from the frontend.
- Stripe line-item detail in the invoice list (showing individual line items vs. a lump sum) is a feature request, not a current capability.
