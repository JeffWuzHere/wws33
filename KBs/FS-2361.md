# Knowledge Base Entry: Prices Visible to Guests and Duplicate Cart Icon with Elementor Theme

## 🎯 Executive Summary
**The Issue:** "Hide Prices for Guests" was not appearing to work because the site was in "Coming Soon" mode, and Elementor's product widgets were displaying a duplicate cart icon for wholesale customers.
**The Fix:** Set the site to Live mode to accurately test the guest price hiding; resolve the duplicate cart icon by placing a custom code snippet (via WP Code plugin) and setting Elementor's "Load Product with AJAX" option to **No**.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** prices still showing to guests, hide prices not working, login to see prices not showing, duplicate cart icon, two cart buttons, Elementor product widget, wholesale price on homepage, coming soon mode
>
> **Steps to solve:**
> 1. **Verify site mode:** If the site is in "Coming Soon" or maintenance mode, the price-hiding behavior may not render correctly through the coming-soon overlay. Ask the customer to temporarily set the site to **Live** and test the guest view in a private/incognito browser.
> 2. **Admin viewing as guest:** Remind the customer that admins viewing the store are treated as regular retail customers — they will see retail prices, not wholesale prices, and the hide-prices-for-guests setting won't apply to logged-in admins.
> 3. **Duplicate cart icon (Elementor):** If the customer is using Elementor and sees two cart icons on product cards, a custom code snippet applied via the WP Code plugin can fix this. After applying the snippet, go to the Elementor widget settings and set **"Load Product with AJAX"** to **No**. The snippet will not work correctly if AJAX loading is enabled.
> 4. **Pricing on homepage vs. product page discrepancies:** These are often caused by the homepage module not reflecting the correct wholesale/guest rules, particularly when using Elementor page builder widgets rather than native WooCommerce shortcodes. The Elementor theme builder may require custom overrides to honor the wholesale pricing visibility rules.

## 💡 Internal Insights (If applicable)
- "Coming Soon" mode from SiteGround (or similar hosting) can make price-hiding features appear broken when tested through the coming-soon overlay. Always confirm the site is Live before diagnosing wholesale price visibility.
- Elementor's "Load Product with AJAX" setting interferes with custom PHP snippets that target WooCommerce product output hooks. Disabling AJAX loading on the widget is a prerequisite for any custom price-display snippet to take effect.
- Homepage vs. product page price discrepancies are a known side effect of Elementor using its own widget rendering pipeline, which may bypass standard WooCommerce filters that Wholesale Prices hooks into.
