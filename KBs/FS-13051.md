# Knowledge Base Entry: Wholesale Order Form Customizations and Wholesale Lead Capture Snippets

## 🎯 Executive Summary
**The Issue:** Multiple confirmed customization requests across Wholesale Order Form and Wholesale Lead Capture: (1) default product quantity on the order form should be 0 instead of 1 with auto-checkbox, (2) "View Cart" popup opens a new tab, (3) admin "New Order" email subject should identify wholesale orders, (4) wholesale registration form shortcode in Elementor, (5) wholesale lead custom fields not editable from My Account, (6) wholesale-only shipping method conflicting with shared shipping methods.
**The Fix:** Each issue has a confirmed working code snippet or native setting. All snippets were tested and confirmed by the customer.

---

## 🛠 Final Resolution (The "2nd Me" Guide)

> [!TIP]
> **Use this solution if a customer asks about:** default quantity zero order form, start at 0 quantity, auto-select checkbox when quantity changed, wwof default qty
>
> **Steps to solve (Default Quantity = 0 with Auto-Checkbox):**
> 1. Add the custom snippet from WPCode library to your child theme's `functions.php` or via WPCode Lite: https://library.wpcode.com/snippet/g5gx48xo/
> 2. This sets all product quantities to 0 on load and automatically ticks the row checkbox when quantity is increased to 1 or more.
> 3. Clear site caches and test on the Order Form page.

---

> [!TIP]
> **Use this solution if a customer asks about:** view cart opens new tab, view cart link new window, cart popup link behavior, change target blank order form
>
> **Steps to solve (View Cart Opens Same Tab):**
> 1. The `target="_blank"` attribute is hardcoded in the Vue.js `AddToCartSuccess` component and cannot be changed with a PHP filter.
> 2. Use a MutationObserver snippet to remove `target="_blank"` when the popup appears. Add this snippet below the default-quantity snippet in `functions.php` or WPCode Lite: https://library.wpcode.com/snippet/r2ml7q65/
> 3. Clear caches and confirm the "View Cart" link routes to the same tab.

---

> [!TIP]
> **Use this solution if a customer asks about:** admin new order email wholesale subject line, customize new order email subject wholesale, woocommerce_email_subject_new_order wholesale, new order notification prefix wholesale
>
> **Steps to solve (Admin "New Order" Email Subject for Wholesale Orders):**
> 1. The `woocommerce_email_subject_new_order` filter is unreliable for this email because WooCommerce triggers it before some plugin data is loaded. Use a direct `$user->roles` check instead.
> 2. Key requirements for the working snippet:
>    - Check `instanceof WC_Order` before running to prevent misfires.
>    - Read roles directly from `$user->roles` (not from plugin-specific meta) — this data is always available at email-send time.
>    - Use `sprintf` to build the subject line cleanly.
> 3. The agent applied a working version to the staging site; check the WP Mail Log plugin for order confirmation.
> 4. If using an email customizer plugin (e.g., Kadence for WooCommerce), set the snippet priority to `99` to run after the email customizer.

> [!TIP]
> **Use this solution if a customer asks about:** registration form shortcode elementor, embed wholesale registration form, wwlc_registration_form shortcode, registration form on custom page
>
> **Steps to solve (Wholesale Registration Form in Elementor):**
> 1. The registration form can be embedded anywhere using the shortcode: `[wwlc_registration_form]`
> 2. In Elementor, add a "Shortcode" widget and paste the shortcode.
> 3. For auto-approval into a specific role on a dedicated page: `[wwlc_registration_form auto_approve=true role=wholesale_customer]`
> 4. Full documentation: https://wholesalesuiteplugin.com/kb/how-to-create-multiple-wholesale-registration-pages/

> [!TIP]
> **Use this solution if a customer asks about:** wholesale shipping method also for retail, shared shipping method wholesale retail, wholesale only shipping hides from retail, duplicate shipping method workaround, restrict shipping by role custom code
>
> **Steps to solve (Role-Restricted Shipping Without Hiding from Others):**
> 1. The native "Wholesale Only Shipping Methods" feature is binary: a mapped shipping method becomes exclusive to wholesale customers and is hidden from retail/guests. There is no native setting to share a method across roles.
> 2. If a shipping provider does not allow creating duplicate methods, use a custom `woocommerce_package_rates` filter snippet instead of the native mapping. Remove all mappings in Wholesale > Settings > Shipping first.
> 3. Example snippet pattern:
> ```php
> add_filter( 'woocommerce_package_rates', function( $rates, $package ) {
>     $role_restricted_methods = array(
>         'Wholesale Flat Rate' => array( 'wholesale_customer' ),
>     );
>     $user_roles = (array) wp_get_current_user()->roles;
>     foreach ( $rates as $rate_id => $rate ) {
>         $allowed_roles = $role_restricted_methods[ $rate->get_label() ] ?? null;
>         if ( $allowed_roles && empty( array_intersect( $allowed_roles, $user_roles ) ) ) {
>             unset( $rates[ $rate_id ] );
>         }
>     }
>     return $rates;
> }, 100, 2 );
> ```
> 4. Methods listed in `$role_restricted_methods` are hidden from users without the specified roles. Methods not listed remain visible to everyone.
> 5. Feature request raised for native "allow wholesale + retail" shipping method sharing.

## 💡 Internal Insights (If applicable)
- The admin "New Order" email timing issue: WooCommerce fires this email before some plugin globals are initialized. The fix is to check `$user->roles` directly rather than using Wholesale Suite-specific plugin data.
- The `woocommerce_package_rates` snippet approach for shipping is the only flexible solution when a third-party shipping provider (e.g., Starshipit) doesn't allow creating duplicate method instances.
- Wholesale Lead custom fields (created in WWLC > Settings > Registration Form) are stored with the `wwlc_cf_` prefix as user meta. When importing users via CSV, use `wwlc_cf_{field_id}` as the column header.
- WWLC lead CSV import meta key reference:
  - Role: `wwlc_unapproved`
  - Active status: `wwlc_is_user_active` = `no`
  - Address fields: `wwlc_address`, `wwlc_address_2`, `wwlc_city`, `wwlc_state`, `wwlc_postcode`, `wwlc_country`, `wwlc_phone`, `wwlc_company_name`
