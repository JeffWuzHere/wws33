# Knowledge Base Entry: Wholesale Price Not Reflected in Elementor Side Cart / Mini Cart

## 🎯 Executive Summary
**The Issue:** Wholesale discounts were not being reflected in an Elementor-powered side/mini cart (though the main cart and checkout pages showed correct prices), because Elementor uses custom cart rendering code that does not natively interact with Wholesale Suite's price hooks.
**The Fix:** Add a code snippet via WP Code that hooks into `woocommerce_cart_item_price` and `woocommerce_before_mini_cart_contents` to force wholesale price recalculation in the mini/side cart; a separate bug where coupon application caused wholesale prices to revert to retail was fixed in the latest Wholesale Prices plugin update.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** Elementor side cart wrong price, mini cart showing retail price, wholesale discount not showing in side cart, cart popup not applying wholesale discount, Elementor cart widget wholesale price, mini cart wholesale
>
> **Steps to solve:**
> 1. Confirm the customer is using Elementor with a side/mini cart widget (not the standard WooCommerce cart page). Verify that the main cart and checkout pages show correct wholesale prices — if those are also wrong, this snippet alone won't solve it.
> 2. Add the following snippet via WP Code (Plugins > WP Code > Add Snippet) or the theme's `functions.php`:
>
> ```php
> add_filter( 'woocommerce_cart_item_price', function( $price, $cart_item, $cart_item_key ) {
>     global $wc_wholesale_prices;
>     $wwp_wholesale_role = $wc_wholesale_prices->wwp_wholesale_roles->getUserWholesaleRole();
>     if ( ! empty( $wwp_wholesale_role ) ) {
>         $wwp_wholesale_prices_instance = new WWP_Wholesale_Prices( array() );
>         $product = wc_get_product( $cart_item['product_id'] );
>         $get_price_html = $product->get_price();
>         if ( $product->is_type( 'variation' ) ) {
>             $parent = wc_get_product( $product->get_parent_id() );
>             $get_price_html = $parent->get_price_html();
>         }
>         return $wwp_wholesale_prices_instance->wholesale_price_html_filter( $get_price_html, $product, $wwp_wholesale_role );
>     }
>     return $price;
> }, 10, 3 );
>
> global $wc_wholesale_prices;
> add_action( 'woocommerce_before_mini_cart_contents', array( $wc_wholesale_prices->wwp_wholesale_prices, 'recalculate_cart_totals' ) );
> ```
>
> 3. After adding the snippet, test by adding a product to the cart as a wholesale user and verifying the side/mini cart and the main cart/checkout all show the same wholesale price.
> 4. If the customer is also experiencing wholesale prices reverting to retail when a coupon is applied, ensure the Wholesale Prices plugin is updated to the latest version — this coupon+wholesale pricing conflict was resolved in a plugin update released in August 2025.
> 5. For the "Hide Quantity Discount Tables" issue (tables still showing after disabling per-product quantity discounts): go to Wholesale > Settings and enable the "Hide Quantity Discount Tables" option. As an alternative, add this CSS via Appearance > Customize > Additional CSS:
> `.order-quantity-based-wholesale-pricing-view.table-view, .qty-based-discount-table-description { display: none !important; }`

## 💡 Internal Insights (If applicable)
- Elementor's side/mini cart renders independently from WooCommerce's standard cart hooks, so Wholesale Suite's price filters do not automatically apply to it. The snippet forces recalculation via `woocommerce_before_mini_cart_contents` and overrides the displayed price via `woocommerce_cart_item_price`.
- The coupon + wholesale pricing conflict (wholesale price reverting to retail when a coupon is used) was a separate confirmed bug, tracked in GitHub issue https://github.com/Rymera-Web-Co/woocommerce-wholesale-prices/issues/800, and fixed in the plugin update released August 2025.
- For role deletion: always check that no user accounts still hold that role before attempting deletion. Also clear the payment gateway mapping for that role in Wholesale Prices > Settings > Payment Gateway.
- If a customer reports intermittent pricing issues that "fix themselves," always check for caching (server, plugin, or browser) as a first step.
