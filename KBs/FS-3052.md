# Knowledge Base Entry: Wholesale Cart Doubles Unit Prices When Using Divi Theme Cart Element

## 🎯 Executive Summary
**The Issue:** When a wholesale customer views the cart, unit prices appear approximately doubled; retail and guest customers are unaffected.
**The Fix:** Replace the Divi theme cart element with the standard WooCommerce cart shortcode (`[woocommerce_cart]`) on the cart page — the agent created a new cart page using the shortcode and the customer confirmed it resolved the issue immediately.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** cart price doubled, wholesale price doubled in cart, price multiplied in cart, Divi cart wrong price, theme cart element wrong price, unit price double for wholesale user
>
> **Steps to solve:**
> 1. Confirm the customer is using the Divi theme and its native Divi cart element/module on the cart page (not the WooCommerce default cart shortcode or block).
> 2. Confirm the doubling only affects wholesale users — retail and guest customers see correct prices. This confirms it is a price filter conflict, not a product data issue.
> 3. Root cause: Divi's cart element uses its own rendering functions and does not invoke standard WooCommerce hooks in the expected order, causing Wholesale Suite's price filters to apply twice (once via Divi's layer, once via WooCommerce's).
> 4. Create or edit the cart page in WordPress. Remove the Divi cart element and replace it with the WooCommerce cart shortcode: `[woocommerce_cart]`.
> 5. Alternatively, advise the customer to contact Divi support, as the issue originates in how Divi's cart element handles WooCommerce price hooks.
> 6. Verify wholesale prices display correctly after the change.

## 💡 Internal Insights (If applicable)
- This is the same class of issue as Elementor cart widget price problems (see FS-3229): any page builder cart element that overrides WooCommerce's standard rendering pipeline can cause Wholesale Suite's price hooks to misfire.
- The doubled price symptom specifically suggests Wholesale Suite's price filter is being applied on top of an already-filtered price — a double-application pattern caused by non-standard hook invocation in the page builder element.
- Quick diagnostic: ask the customer to test with a plain page containing only `[woocommerce_cart]` shortcode. If prices are correct there, the page builder cart element is the cause.
- This issue is not a bug in Wholesale Suite — it is a theme/page builder compatibility issue. Divi support may be able to address it from their side as well.
