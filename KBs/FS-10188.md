# Knowledge Base Entry: Order Notification Emails Not Sending After WordPress Update

## 🎯 Executive Summary
**The Issue:** Admin order notification emails stopped sending after a WordPress update, and manually triggering "Resend New Order" emails also failed.
**The Fix:** The root cause was a broken server-level PHP mail function; configuring an SMTP plugin (e.g., WP Mail SMTP) with valid SMTP credentials restores email delivery.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** order emails not sending, new order notification not received, resend order email not working, WordPress emails stopped after update, WooCommerce not sending emails
>
> **Steps to solve:**
> 1. Confirm the issue is server/SMTP related, not plugin-specific: test by installing WP Mail SMTP and configuring it with valid SMTP credentials (e.g., Gmail SMTP). If emails send successfully, the server's native PHP mail function is the culprit.
> 2. Instruct the customer to set up their own SMTP configuration in WP Mail SMTP (or equivalent) using their own email provider credentials — not the support agent's.
> 3. If the customer does not have SMTP credentials, direct them to contact their hosting provider to investigate why the server's PHP mail function is failing.
> 4. Share these resources: https://wholesalesuiteplugin.com/kb/troubleshoot-emails-not-sending/ and https://wpmailsmtp.com/how-to-fix-could-not-instantiate-mail-function-in-wordpress/

When I go to an order and click "Resend New Order Information," I still don't receive the email. The shipping integration is still receiving order data, so something is working — just not the admin email notifications.

**[SUPPORT_AGENT]:**
I accessed the site and investigated the issue. The problem is with the server or SMTP configuration — it is not caused by Wholesale Suite. I tested this by activating WP Mail SMTP and entering SMTP credentials, and emails sent successfully. I then disabled WP Mail SMTP (since it was configured with my credentials). To fix this permanently, you need to either configure WP Mail SMTP with your own email provider's SMTP credentials, or contact your hosting provider to find out why the native PHP mail function is failing.

Note: this is a common WordPress issue — WordPress's native mail function has poor deliverability, and SMTP plugins are the standard fix.

## 💡 Internal Insights (If applicable)
- This issue is not caused by Wholesale Suite or WooCommerce. It is a server-level PHP mail failure. Do not spend time debugging plugin conflicts.
- The key diagnostic test: install WP Mail SMTP, configure with any working SMTP credentials, and test. If it works, the server's `mail()` function is broken.
- The customer's shipping integration continuing to work (e.g., via API) does not mean email is functional — shipping plugins typically use direct API calls, not PHP mail.
