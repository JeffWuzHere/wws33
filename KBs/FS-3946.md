# Knowledge Base Entry: _load_textdomain_just_in_time Warning on WordPress 6.8 with Wholesale Order Form and Wholesale Prices Premium

## 🎯 Executive Summary
**The Issue:** After updating to WordPress 6.8+, sites using Wholesale Order Form and/or Wholesale Prices Premium display a `_load_textdomain_just_in_time was called incorrectly` PHP notice because translation loading is triggered before the `init` hook.
**The Fix:** Update Wholesale Order Form to v3.0.7 or later, which contains the official fix; the warning can be suppressed in the meantime with a code snippet.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** `_load_textdomain_just_in_time`, translation loading too early, WordPress 6.7 or 6.8 notices, load_textdomain warning, wholesale plugin PHP notice, WP 6.8 compatibility
>
> **Steps to solve:**
> 1. Confirm whether the customer is running WordPress 6.7.0 or later — this is when the notice was introduced.
> 2. Instruct the customer to update Wholesale Order Form to v3.0.7 or later (the fix is included in that release).
> 3. If the customer is also seeing the notice from Wholesale Prices Premium, check for a corresponding update; advise they update to the latest version available.
> 4. If the customer cannot update immediately and needs a temporary workaround, provide the suppression code snippet via WP Code plugin or child theme `functions.php` (see Internal Insights below for the GitHub Gist reference pattern).
> 5. Note: the notice is cosmetic and does not affect store functionality or pricing behavior — reassure the customer if they are alarmed.

## 💡 Internal Insights (If applicable)
- The root cause traced in the stack: `License_Manager->__construct()` calls `WWOF::get_current_plugin_version()` → `WWOF::get_current_plugin_data()` → `get_plugin_data()` → triggers translation loading before `init`. This was too early in the WordPress load cycle.
- GitHub issues logged: woocommerce-wholesale-order-form#835 and woocommerce-wholesale-prices-premium#1423.
- The fix was shipped in Wholesale Order Form v3.0.7. Check if a corresponding WWPP release is available at time of handling.
- The notice does not affect pricing or store functionality — it is purely a PHP debug notice visible in Query Monitor or when `WP_DEBUG` is enabled.
