# Knowledge Base Entry: Slow Product Save Times with Many Wholesale Roles

## 🎯 Executive Summary
**The Issue:** Saving a product with variations takes 10–12 seconds when 33+ wholesale roles are configured, causing concern that the plugin has a memory leak or performance regression.
**The Fix:** This is expected behavior — the save process runs two sequential operations (admin.ajax processes all role data ~5s, then the post save writes to the database ~5s), and the total time scales with the number of roles configured.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** slow product save, product editor timeout, variations taking long to save, wholesale role performance, AJAX 504 on product save, save delay with many roles
>
> **Steps to solve:**
> 1. Ask the customer to perform a basic debug (disable all plugins except WooCommerce and Wholesale Suite, switch to a default theme) to rule out third-party conflicts.
> 2. Request access to a staging site to observe the save time directly.
> 3. Visit a product with variations in the WP admin and time the save. Observe both the `admin.ajax` and `post` requests in the browser network tab.
> 4. If `admin.ajax` takes ~5 seconds and the `post` save takes ~5 seconds (total ~10–12s), confirm this is **within normal range** for 30+ wholesale roles. The admin.ajax call iterates through all wholesale role data before the database write occurs.
> 5. Only escalate if save time exceeds 1+ minute, which would indicate a genuine performance problem.

## 📝 Sanitized Interaction History

**[CUSTOMER]:**
After upgrading to Wholesale Prices Base v2.2.3 and Wholesale Prices Premium v2.0.4, we get an AJAX 504 error when updating product variations. We have about 33 user roles with 4 variations per product. The older version (v1.27.7) handled this without errors. We suspect a memory leak because the newer version requires us to increase PHP resources just to function.

**[SUPPORT_AGENT]:**
The error logs pointed toward Jetpack and Yoast, not our plugin. Please disable those plugins one at a time, run a basic debug (only WooCommerce, Wholesale plugins, and a default theme), and share admin access to a staging site if the issue persists.

**[CUSTOMER]:**
Performed the basic debug — disabled all plugins except WooCommerce, Wholesale Prices, and Wholesale Prices Premium, used the Twenty Twenty-Five theme. Saving variations takes about 10 seconds — the first variation edits quickly but subsequent ones are slow. Full product save takes ~12 seconds. We are on a dedicated server with increased PHP resources.

**[SUPPORT_AGENT] (after visiting staging site):**
I accessed the staging site and confirmed the behavior via screen recording. Everything is working as intended. The 10–12 second total save time is composed of two parts:
- **admin.ajax (~5 seconds):** Iterates through all wholesale role data. This is proportional to the number of roles configured — with 33 roles this is within normal range.
- **post save (~5 seconds):** The actual database write of the processed data.

This is only a concern if save time reaches 1+ minute. The current behavior is expected.

## 💡 Internal Insights (If applicable)
- The original 504 error was likely a server timeout triggered by the increased load of the new plugin version combined with Jetpack/Yoast overhead — not a plugin bug.
- The customer's comparison to the older version is valid: the newer version does more processing per role, but this is by design and within acceptable thresholds.
- If a customer has 100+ roles and save times become unacceptable (minutes, not seconds), that would warrant a developer escalation for a performance review.
- The `tc_tk` cookie mentioned by the customer is a TikTok Pixel cookie from a third-party analytics script — unrelated to the plugin.
