# Knowledge Base Entry: Wholesale Price Not Showing Correctly in Elementor Header Cart Widget

## 🎯 Executive Summary
**The Issue:** When a wholesale customer adds a product to the cart, the price shown in the Elementor header cart/mini-cart widget displays an incorrect price (e.g., the standard wholesale price rather than the role-specific price), while the cart page total and single product page both show the correct price.
**The Fix:** Add a custom code snippet (via WP Code plugin or `functions.php`) that forces the Elementor cart widget to use the correct WooCommerce price functions; the agent applied snippet ID 16 on the staging site and the customer confirmed it resolved the issue.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** Elementor cart price wrong, mini cart header wrong price, cart widget showing incorrect wholesale price, Elementor header cart displaying retail price, price mismatch in cart dropdown
>
> **Steps to solve:**
> 1. Confirm the customer is using Elementor and its built-in cart/mini-cart widget in the header (not a standard WooCommerce cart block or shortcode).
> 2. Confirm that the cart page itself and the single product page show the correct wholesale price — the issue is isolated to the Elementor widget.
> 3. The root cause: Elementor's cart widget uses its own rendering functions rather than standard WooCommerce hooks, which bypasses Wholesale Suite's price filters.
> 4. Apply the appropriate code snippet to make the Elementor cart widget respect wholesale pricing. Add the snippet via the WP Code plugin or in the child theme's `functions.php`.
> 5. Ask the customer to test after the snippet is applied and confirm the header cart widget now shows the correct role-specific price.

## 💡 Internal Insights (If applicable)
- This is a known compatibility pattern: any page builder (Elementor, Divi, etc.) that renders cart/mini-cart using its own widget/module — rather than WooCommerce's native shortcode `[woocommerce_cart]` or block — may bypass Wholesale Suite price filters.
- The fix is a code snippet, not a plugin setting. The exact snippet was applied server-side during the support session; when handling similar tickets, search internal snippet library or check the WP Code plugin on the customer's site for reference.
- The same class of issue applies to Divi (see FS-3052) — when in doubt, test with the WooCommerce default cart shortcode to isolate whether the page builder widget is the cause.
