# Knowledge Base Entry: Wholesale Pricing Not Applied When Creating or Editing Orders in WooCommerce Admin

## 🎯 Executive Summary
**The Issue:** When an admin creates or edits an order from the WooCommerce backend for a wholesale customer, items are added at retail price rather than wholesale price. The "Apply Wholesale Pricing" option exists but category/percentage discounts are not supported in backend orders — only fixed per-product wholesale prices are applied.
**The Fix:** Enable the "Apply Wholesale Pricing" option on the backend order and remove/re-add items. For stores using category-level percentage discounts (not fixed prices), the only supported workaround is to use the User Switching plugin to place the order as the wholesale customer from the frontend.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** manual order wholesale price, backend order retail price, admin creating order for wholesale customer, order edit wholesale pricing, wholesale price not applying in admin, category discount not working in manual order
>
> **Steps to solve:**
> 1. In the WooCommerce order edit screen, look for the **"Apply Wholesale Pricing"** checkbox/option and enable it.
> 2. Set the order status to **Pending** or **On Hold** (required for the option to take effect).
> 3. **Remove the existing line items** and re-add the products. Items added before the "Apply Wholesale Pricing" option was enabled will retain their retail price — this is a WooCommerce limitation.
> 4. After re-adding items, click **Recalculate** to update the order totals.
> 5. **If the customer uses category-level percentage discounts** (not fixed per-product wholesale prices): the "Apply Wholesale Pricing" option will NOT apply these discounts in the backend. In this case, use the **User Switching** plugin (https://wordpress.org/plugins/user-switching/) to temporarily switch to the wholesale customer's account and place the order from the frontend, where percentage discounts apply correctly.

## 📝 Sanitized Interaction History

**[CUSTOMER]:**
When I change an order placed by a wholesale customer (delete the original products and add them back), the items use the retail price instead of the preset wholesale price. I want wholesale pricing to apply automatically when adding items to an order for a wholesale customer.

**[SUPPORT_AGENT]:**
On the order edit screen, enable the "Apply Wholesale Pricing" option and set the order to Pending or On Hold. Then delete existing items and re-add them — existing items that were added at retail won't retroactively change price. After re-adding, click Recalculate.

**[CUSTOMER]:**
I tried that and the price is still retail. I set up wholesale pricing using a universal category discount (e.g., 20% off all items in a category), not per-product prices.

**[SUPPORT_AGENT]:**
Understood — category/general discounts are currently not supported in backend manual orders. Only fixed per-product wholesale prices are applied. This is a known limitation (feature request logged). The workaround is to use the User Switching plugin to log in as the wholesale customer and place the order from the frontend, where category discounts apply correctly.

## 💡 Internal Insights (If applicable)
- There are two distinct wholesale pricing methods: (1) fixed per-product price, and (2) category/role percentage discount. Only method 1 works in backend manual orders.
- The "Apply Wholesale Pricing" checkbox on orders only triggers fixed wholesale prices — it does not trigger the general discount calculation.
- Feature request for category discount support in manual orders: https://github.com/Rymera-Web-Co/woocommerce-wholesale-prices-premium/issues/1387
- Separately in this ticket: a "spinning/recalculate" error on a specific order was caused by bad tax data (float * string type error in WooCommerce). The agent fixed it by resetting the tax values on the order. This is a WooCommerce data integrity issue, not a Wholesale Suite bug.
- User Switching plugin: https://wordpress.org/plugins/user-switching/
