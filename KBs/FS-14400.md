# Knowledge Base Entry: "Sorry You Are Not Allowed to Access This Page" Error When Accessing Lead Capture Settings

## 🎯 Executive Summary
**The Issue:** Customers running a significantly outdated version of Wholesale Lead Capture receive a "Sorry, you are not allowed to access this page" error when attempting to access the plugin settings or license page, because the settings page URL structure has changed in newer versions and the old version cannot resolve it.
**The Fix:** Manually install the latest version of Wholesale Lead Capture (bypassing the in-plugin updater), then update the license key — after which all settings pages become accessible at their new location.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** not allowed to access this page wholesale lead, lead capture settings 403, license page blocked, cannot update lead capture plugin, wholesale lead settings missing
>
> **Steps to solve:**
> 1. Confirm the customer is running a significantly outdated version of Wholesale Lead Capture (the error occurs because the plugin's admin page slug/URL has changed in a newer version and the outdated version cannot route to it).
> 2. Download the latest version of Wholesale Lead Capture from the customer's account at wholesalesuiteplugin.com/my-account/downloads/.
> 3. Install the plugin manually via **WordPress Admin > Plugins > Add New > Upload Plugin**, uploading the downloaded zip. Choose **Replace current with uploaded** when prompted.
> 4. Once the latest version is active, go to **Wholesale > Licenses > Wholesale Lead** and enter/update the license key.
> 5. Confirm the customer can now access all Lead Capture settings. Note: settings have moved to **Wholesale > Settings > Wholesale Lead > Registration** (URL: `wp-admin/admin.php?page=wholesale-settings&tab=wholesale_lead&tab-child=registration`).
> 6. Advise the customer to review registration form settings to ensure their configuration is intact after the update.

## 💡 Internal Insights (If applicable)
- The root cause is that very old versions of WWLC register admin page slugs that no longer exist in current versions. The settings page URL changed when WWLC settings were consolidated under the unified Wholesale Suite settings panel.
- The in-plugin updater cannot run because it requires accessing the license page — which is itself blocked. This creates a catch-22 that requires manual installation of the latest version.
- After a major version update, always flag to the customer that settings may need to be re-verified, particularly the registration form fields and approval workflow settings.
