# Knowledge Base Entry: Wholesale Order Form Shows "Insufficient Stock" Despite Products Being In Stock

## 🎯 Executive Summary
**The Issue:** Specific products display an "insufficient stock" message on the Wholesale Order Form even though WooCommerce admin shows them as In Stock with available quantity. The issue affects only a subset of newly created products while thousands of others work correctly.
**The Fix:** The "insufficient stock" message in Wholesale Suite is triggered when the product's available stock quantity is lower than the minimum order quantity (MOQ) configured for the wholesale role — either at the product level or in the global settings. Checking and correcting the MOQ resolves the message.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** insufficient stock order form, product shows insufficient stock but is in stock, wholesale order form stock error, stock message wrong wholesale, newly created products insufficient stock, minimum order quantity stock error
>
> **Steps to solve:**
> 1. Navigate to the affected product's edit page in WooCommerce admin.
> 2. Check the **Wholesale Prices** tab on the product editor — look for a "Minimum Order Quantity" field set for the wholesale role. If this value exceeds the current stock quantity, the "insufficient stock" message will display.
> 3. Also check the global minimum order quantity setting: Wholesale > Settings > General — verify the global MOQ is not set higher than the product's stock level.
> 4. Lower the MOQ to a value equal to or less than the available stock quantity, or increase the stock quantity to meet the MOQ requirement.
> 5. The issue appearing only on newly created products typically indicates those products were created without adjusting the MOQ, while older products either have no MOQ set or have MOQs that match their stock levels.

## 💡 Internal Insights (If applicable)
- The "insufficient stock" message is not a WooCommerce stock status message — it is generated by Wholesale Suite when the stock quantity is below the configured MOQ for the role. This is a common source of confusion because WooCommerce admin shows the product as "In Stock."
- The issue disproportionately affects newly created products when a global MOQ is set and the new products haven't been given stock quantities above that threshold.
- If no MOQ is intentionally set, the global value in Wholesale > Settings > General may have been set inadvertently and should be cleared or set to 1.
