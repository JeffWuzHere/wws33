# Knowledge Base Entry: Products Not Showing for Guest/Non-Logged-In Users

## 🎯 Executive Summary
**The Issue:** Guest visitors see "no products found" on the shop page while wholesale-role users can browse products normally — caused by a combination of a missing shop page setting, a broken Divi shop page, and per-product wholesale-only visibility filters.
**The Fix:** Set the WooCommerce shop page correctly, create a non-Divi shop page if needed, and remove product-level wholesale-only visibility restrictions so all visitors can browse the catalog.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** no products found for guests, shop page empty for non-logged-in users, products only visible when logged in as wholesale, guest shop page blank, imported products not visible
>
> **Steps to solve:**
> 1. Go to **WooCommerce > Settings > Products > General** and confirm that a **Shop page** is set. If blank, assign any standard WooCommerce shop page here.
> 2. If the shop page was built with a page-builder (e.g. Divi), test by creating a plain new page (no Divi builder blocks) and assign it as the shop page — Divi shop pages can become inaccessible or render without products.
> 3. Check product-level visibility: in **Wholesale > Settings > Wholesale Prices**, confirm that products are NOT set to show only to wholesale roles unless that is intentional. Per-product, the `wwpp_product_wholesale_visibility_filter` meta must be set to `all` (not a specific role) for guests to see the product.
> 4. For bulk correction on 100+ products: export products via WooCommerce CSV export, set the `wwpp_product_wholesale_visibility_filter` column to `all` for all rows, then re-import. Alternatively use the **Re-Initialize Product Visibility** option under **Wholesale > Settings > Help**.
> 5. Clear all plugin and server caches after making changes. After importing products, always run **Re-Initialize Product Visibility** to rebuild visibility metadata.

**[SUPPORT_AGENT]:**
Visited the customer's site directly. Found three issues:

## 💡 Internal Insights (If applicable)
- When a customer imports products via CSV, the wholesale visibility metadata may not be set correctly on imported items — always prompt them to run Re-Initialize Product Visibility after an import.
- Divi's shop builder can prevent the standard WooCommerce shop page from working. If a customer uses Divi and reports a blank shop, recommend testing with a plain (non-Divi) page assigned as the WooCommerce shop page.
- The hide-prices feature (Wholesale > Settings > Wholesale Prices > Hide Price and Add to Cart) only works if product visibility is set to show products to all users — if visibility is restricted to wholesale roles, guests are redirected/blocked entirely before seeing any product, so prices never appear.
