# Knowledge Base Entry: Using Wholesale Suite with WooCommerce REST API for Headless / API-Only Orders

## 🎯 Executive Summary
**The Issue:** A customer using the WooCommerce REST API to submit orders (no checkout page) needed to know whether wholesale pricing would be applied automatically and whether per-customer (individual) pricing was supported.
**The Fix:** Wholesale pricing is NOT automatically applied when prices are fetched via the standard WooCommerce REST API; instead, the Wholesale Suite API must be used to retrieve the correct wholesale price for a role, and that price must be explicitly set on the WooCommerce order via the REST API. Per-customer pricing is not natively supported but can be approximated via per-user general discount overrides.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** wholesale pricing REST API, WooCommerce API wholesale price, headless WooCommerce wholesale, API orders wholesale discount, per-customer pricing, individual customer pricing, wholesale price not applied via API
>
> **Steps to solve:**
> 1. **Per-customer vs. role-based pricing:** Explain that Wholesale Suite pricing is role-based, not per-customer by default. However, a per-user override is available: configure a General Discount for the wholesale role, then override the discount amount on a per-user basis via the user profile settings. See: https://wholesalesuiteplugin.com/kb/overriding-wholesale-settings-per-user/
>
> 2. **WC REST API + wholesale pricing:** When an order is submitted via the standard WooCommerce REST API, wholesale prices are NOT automatically applied — WooCommerce uses the standard product price. The correct workflow is:
>    - Step 1: Use the Wholesale Suite API to fetch the wholesale price for the product and the customer's wholesale role.
>    - Step 2: In the WooCommerce REST API order creation request, explicitly set the line item price to the wholesale price retrieved in Step 1.
>
> 3. Reference the Wholesale Suite API documentation for retrieving wholesale prices: the internal ClickUp doc linked in the resolution provides the API endpoint details (share with the customer as appropriate).
>
> 4. If the customer is unsure which wholesale role a given user belongs to, they can retrieve this via the WordPress user meta or via the Wholesale Suite user management UI.

## 📝 Sanitized Interaction History

**[CUSTOMER]:**
With Wholesale Suite, does it allow for pricing based on individual customers? Or does it need to be done via user roles? Also, for our specific use case, orders will come across via the WC REST API — so there is no checkout page. Will the pricing options work via API?

**[SUPPORT_AGENT]:**
Pricing is based on customer roles; it is not currently possible to set wholesale prices for individual customers directly. However, it can be achieved by using General Wholesale Discounts for your store and then overriding the value on a per-user basis: https://wholesalesuiteplugin.com/kb/overriding-wholesale-settings-per-user/

Regarding the WooCommerce REST API: if you are fetching the price directly via WooCommerce, the wholesale price will not be applied automatically. However, we have our own API and a workaround:
1. Use the Wholesale Suite API to fetch the wholesale price of the product for the wholesale role/customer.
2. In the WC REST API order creation call, set the price of the product to the fetched wholesale price.

## 💡 Internal Insights (If applicable)
- This is a common question from developers building headless or API-driven WooCommerce stores where there is no traditional browser-based checkout session. Wholesale pricing hooks into WooCommerce's session/cart layer, which does not exist in REST API calls.
- The two-step pattern (fetch via Wholesale Suite API → set explicitly in WC REST API) is the only supported approach for this use case.
- Per-user discount overrides are documented at https://wholesalesuiteplugin.com/kb/overriding-wholesale-settings-per-user/ — this is the closest approximation to per-customer pricing within the current feature set.
- If a customer needs true per-customer pricing (each customer has a unique price independent of any role), this is not currently supported and would require a feature request or custom development.
