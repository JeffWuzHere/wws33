# Knowledge Base Entry: WooPayments Not Supported by Wholesale Payment Gateway Restrictions — Custom Snippet Workaround

## 🎯 Executive Summary
**The Issue:** Wholesale Suite's built-in Payment Gateway Mapping/Restriction feature does not support WooPayments (or its sub-gateways), so wholesale customers still see all available payment options at checkout regardless of the restriction settings.
**The Fix:** A custom PHP snippet added to functions.php (or via WPCode plugin) can programmatically restrict payment gateways for wholesale customers on the frontend; the customer confirmed this workaround functions correctly on their live site.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** WooPayments gateway restriction not working, wholesale payment gateway mapping WooPayments, restrict checkout payment methods wholesale, wholesale customers see all payment options, payment gateway filter not working with WooPayments
>
> **Steps to solve:**
> 1. Confirm the customer is using WooPayments as their gateway and that the built-in Wholesale Prices Premium Payment Gateway Mapping is not restricting it.
> 2. Inform the customer that WooPayments is not currently fully supported by the built-in gateway restriction feature — this is a known limitation with a feature request raised internally.
> 3. Provide the following custom PHP snippet. The customer should add it to their theme's functions.php or via the WPCode plugin. Modify the `$allowed_gateways` array to include only the gateway IDs that wholesale customers should be able to use (e.g., `woocommerce_payments` for credit card via WooPayments):
>
> ```php
> add_filter( 'woocommerce_available_payment_gateways', 'restrict_payment_gateways_for_wholesale' );
> function restrict_payment_gateways_for_wholesale( $available_gateways ) {
>     if ( is_admin() ) return $available_gateways;
>
>     // Check if the current user has a wholesale role
>     $user = wp_get_current_user();
>     $wholesale_roles = array( 'wholesale_customer' ); // adjust role slugs as needed
>
>     $is_wholesale = false;
>     foreach ( $wholesale_roles as $role ) {
>         if ( in_array( $role, (array) $user->roles ) ) {
>             $is_wholesale = true;
>             break;
>         }
>     }
>
>     if ( $is_wholesale ) {
>         // List only the gateway IDs allowed for wholesale customers
>         $allowed_gateways = array( 'woocommerce_payments' ); // credit card via WooPayments
>         foreach ( $available_gateways as $id => $gateway ) {
>             if ( ! in_array( $id, $allowed_gateways ) ) {
>                 unset( $available_gateways[ $id ] );
>             }
>         }
>     }
>
>     return $available_gateways;
> }
> ```
>
> 4. Instruct the customer to identify the correct WooPayments gateway IDs by inspecting the checkout page or checking WooCommerce > Settings > Payments — the gateway IDs are shown in the URL when editing each gateway.
> 5. Raise a feature request for native WooPayments support in the gateway restriction feature if not already logged.

## 💡 Internal Insights (If applicable)
- WooPayments registers multiple sub-gateway IDs (e.g., `woocommerce_payments`, `woocommerce_payments_affirm`, etc.) which the built-in mapping feature does not enumerate correctly — hence it cannot restrict them.
- The `woocommerce_available_payment_gateways` filter is the correct WordPress/WooCommerce hook to use for this workaround.
- A feature request has been raised internally to add native WooPayments support to the Payment Gateway Mapping feature.
- Always ask customers to test on a staging site before applying custom code snippets to live sites.
