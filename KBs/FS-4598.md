# Knowledge Base Entry: Wholesale Payments Invoice Gateway Not Saving PO Numbers

## 🎯 Executive Summary
**The Issue:** When using the Wholesale Payments Invoice Gateway, PO (purchase order) numbers entered by customers at checkout were not being saved — the field appeared but submitted values were discarded.
**The Fix:** A hotfix build of the Wholesale Payments plugin resolves this issue; apply the hotfix while awaiting the next official public release.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** PO number not saving, purchase order number not recorded, invoice gateway PO field not working, WooCommerce Wholesale Payments PO, invoice gateway save issue, PO field blank after checkout
>
> **Steps to solve:**
> 1. Confirm the customer is using the Wholesale Payments plugin with the Invoice Gateway payment method.
> 2. Confirm the symptom: customer fills in a PO number field at checkout, but after the order is placed the PO number is not recorded on the order.
> 3. Check whether an official plugin update has been released that addresses this issue — if the current version is later than the date of this article (July 2025), update Wholesale Payments first and retest.
> 4. If the issue persists on the current release, contact the Wholesale Suite support team to obtain the latest hotfix build. Before applying, ensure a full site backup is in place.
> 5. Install the hotfix by uploading the plugin zip via Plugins > Add New > Upload Plugin, overwriting the existing version.
> 6. Retest PO number saving by placing a test order through the Invoice Gateway checkout.
