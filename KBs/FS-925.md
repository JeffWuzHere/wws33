# Knowledge Base Entry: Apply Cart Subtotal Discount to Wholesale Products Only (Exclude Retail Products)

## 🎯 Executive Summary
**The Issue:** A customer needed the Cart Subtotal Price Discount to apply only to wholesale-priced products in a mixed cart, leaving retail-priced products unaffected.
**The Fix:** This is now a native setting in Wholesale Prices Premium v2.0.6: navigate to Wholesale > Discounts and enable **"Apply discounts to wholesale products only."**

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** cart subtotal discount wholesale only, exclude retail from discount, mixed cart discount, subtotal discount not applying correctly, discount on wholesale products only, cart price discount retail excluded
>
> **Steps to solve (using native setting — WWPP v2.0.6+):**
> 1. Update Wholesale Prices Premium to **v2.0.6 or later**.
> 2. In the WordPress admin, navigate to **Wholesale > Discounts**.
> 3. Enable the option **"Apply discounts to wholesale products only."**
> 4. Save settings. The Cart Subtotal Price Discount will now only count the subtotal of wholesale-priced products and apply the percentage discount to those products only.
>
> **Steps to solve (code snippet — for older plugin versions):**
> 1. Add the following snippet to the theme's `functions.php` or via the WP Code plugin:
> ```php
> function wws_modify_cart_subtotal_discount($discount_data) {
>     global $wc_wholesale_prices;
>     if (!isset($wc_wholesale_prices) || empty(WC()->cart)) {
>         return $discount_data;
>     }
>     $wholesale_role = $wc_wholesale_prices->wwp_wholesale_roles->getUserWholesaleRole();
>     if (empty($wholesale_role)) {
>         return $discount_data;
>     }
>     $wholesale_subtotal = 0;
>     foreach (WC()->cart->get_cart() as $cart_item) {
>         $product_id = $cart_item['data']->get_id();
>         $wholesale_price = $wc_wholesale_prices->wwp_wholesale_prices::get_product_wholesale_price_on_cart(
>             $product_id, $wholesale_role, $cart_item, WC()->cart
>         );
>         if ($wholesale_price) {
>             // accumulate wholesale subtotal only
>             $wholesale_subtotal += $wholesale_price * $cart_item['quantity'];
>         }
>     }
>     // Override the subtotal used in the discount calculation
>     $discount_data['cart_subtotal'] = $wholesale_subtotal;
>     return $discount_data;
> }
> ```
> 2. This filters the cart subtotal passed to the discount engine to only include line items that have a wholesale price assigned.

Confirmed understanding of the requirement: the cart subtotal discount should only count and apply to products that have a wholesale price, leaving retail-priced items unaffected. **[SUPPORT_AGENT] (v2.0.6 release notification):**
Wholesale Prices Premium v2.0.6 introduces a native setting for this: navigate to **Wholesale > Discounts** and enable **"Apply discounts to wholesale products only."** Update to v2.0.6 to use this feature without custom code.

## 💡 Internal Insights (If applicable)
- Prior to WWPP v2.0.6, the only way to restrict cart subtotal discounts to wholesale products was via the custom PHP snippet provided. Customers on older versions who have the snippet installed can remove it after upgrading to v2.0.6 and enabling the native setting.
- The native setting in v2.0.6 is found at Wholesale > Discounts, not under Wholesale > Settings, which may cause confusion if a customer is looking in the wrong place.
- The use case driving this request is a store with a mixed product catalog where some products are sold wholesale (with a wholesale price set) and others are always sold at retail price to all customers.
