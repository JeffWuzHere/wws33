# Knowledge Base Entry: Incorrect Cart Price When Wholesale Minimum Order Not Met — Caused by Custom Theme Snippet

## 🎯 Executive Summary
**The Issue:** Wholesale customers who did not meet the minimum order threshold were seeing an unexpected intermediate price in the cart (neither the correct wholesale price nor the correct retail price), caused by a custom price-override snippet in the site's Porto child theme `functions.php` that was applying a flat percentage discount on top of WooCommerce's cart totals, interfering with Wholesale Suite's minimum order logic.
**The Fix:** Remove or disable the conflicting custom snippet from the child theme; Wholesale Suite's built-in minimum order settings then behave correctly without any additional code.

---

## 🛠 Final Resolution (The "2nd Me" Guide)
> [!TIP]
> **Use this solution if a customer asks about:** wrong price when minimum order not met, unexpected price in cart, cart showing neither retail nor wholesale price, price calculation error wholesale minimum, child theme snippet conflict, woocommerce_before_calculate_totals conflict, minimum order threshold wrong price
>
> **Steps to solve:**
> 1. Confirm the specific symptom: wholesale customers see a price that is neither the set wholesale price nor the standard retail price when the minimum order total is not met.
> 2. Before debugging Wholesale Suite settings, check the site's active theme (especially child themes) for custom snippets hooking into `woocommerce_before_calculate_totals`. This hook is commonly used for custom price overrides and will conflict with Wholesale Suite's price management.
> 3. To locate: go to Appearance > Theme Editor > `functions.php` on the active child theme (or check via WP Code / Code Snippets plugin). Search for `woocommerce_before_calculate_totals`, `set_price`, or `get_regular_price`.
> 4. If a custom price-calculation snippet is found, comment it out or remove it, then test the cart behavior again as a wholesale customer — both above and below the minimum order total.
> 5. For properly blocking wholesale customers from placing orders below the minimum: use Wholesale Suite's built-in feature rather than custom snippets. See: https://wholesalesuiteplugin.com/kb/how-to-stop-wholesale-customers-from-purchasing-until-they-reach-the-minimum-requirements/
> 6. If the customer also reports that the latest Wholesale Prices Free update (v2.2.4) caused wholesale prices to revert to retail when a coupon is applied, advise them to update to the latest version of Wholesale Prices — this was confirmed as a bug introduced in v2.2.4 and fixed in a subsequent release.

## 📝 Sanitized Interaction History

**[CUSTOMER]:**
We have two wholesale roles with minimum order thresholds. When a customer in the DEALERS role meets the minimum (€122 inc. VAT), the correct wholesale price (€7.08) is applied. But when they don't meet the minimum, instead of the retail price (€10.90), an unexpected price of €8.72 appears on orders. We cannot identify where this intermediate price originates. We've also set a minimum for the VIP role which always worked. Credentials provided for staging.

**[SUPPORT_AGENT]:**
After accessing the staging site, the root cause is a custom snippet in the Porto child theme's `functions.php` that hooks into `woocommerce_before_calculate_totals`. The function (`allinea_prezzi_carrello`) applies a flat 20% discount off the regular price for wholesale users, which conflicts with Wholesale Suite's minimum order price logic. This override was being applied to cart items regardless of whether the minimum order total was met, producing the unexpected intermediate price. The snippet has been commented out on the staging site for review.

**[CUSTOMER]:**
Confirmed — removing the snippet from the live site fixed the issue. The snippet was originally added to align prices with an Advanced Coupons reward system.

*(Follow-up: Separate issue — plugin update v2.2.4 regression)*

**[CUSTOMER]:**
After updating Wholesale Prices Free to v2.2.4 on staging, wholesale groups now display retail prices in the cart instead of their discounted prices once the threshold is reached. Rolling back to v2.2.3 fixed the live site. This is a v2.2.4 bug.

**[SUPPORT_AGENT]:**
Confirmed the issue on the staging site. Root cause identified: when an Advanced Coupons coupon is active, v2.2.4 causes the cart to use retail price instead of the wholesale price. This is a bug — raised with developers. Do not update Wholesale Prices Free to v2.2.4 on your live site. A hotfix has been issued; official fix released subsequently in the next plugin version.

**[SUPPORT_AGENT]:**
The issue affecting cart/checkout when applying coupons to wholesale pricing has been officially resolved. Please update Wholesale Prices to the latest version.

## 💡 Internal Insights (If applicable)
- Custom snippets using `woocommerce_before_calculate_totals` with `$product->set_price()` will override Wholesale Suite's price management, including minimum order threshold behavior. Always check child theme `functions.php` and any snippet plugins when cart prices are unexpectedly wrong.
- The v2.2.4 coupon regression bug (wholesale price reverting to retail when a coupon is active) is tracked at GitHub issue https://github.com/Rymera-Web-Co/woocommerce-wholesale-prices/issues/800.
- Wholesale Suite has a native minimum order enforcement feature that does not require custom code: https://wholesalesuiteplugin.com/kb/how-to-stop-wholesale-customers-from-purchasing-until-they-reach-the-minimum-requirements/
- Porto theme's built-in free shipping bar (yellow banner showing progress to free shipping) is a Porto theme feature, not Wholesale Suite — configure it in Porto Settings, not Wholesale Suite settings.
